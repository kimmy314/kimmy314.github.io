Section Analysis Tool

Tool I designed and developed to increase the efficiency of analyzing sections.

This tool was created in Python using MVC design pattern using OOP practices.

The tool takes in simple polygons and calculates the principal moments of inertia along with the theta relative to the graph x axis. The tool also is able to calculate stresses and locate the minimum and maximum stress points for a given load. This allows for users to quickly identify the critical locations and determine if a section fails based on an input load.

A little bit about the design details. An abtract shape class has fields which contain geometry details such as Xcg. The class calculates generic CG positions as well and declares a calculateMOIAboutCG function and defines a calculateMOIAboutZed

The calculateMOIAboutZed is defined within shapes since once a moment of inertia about the CG is found, the MOI about the 0,0 position is simply IXX + A_total * Xcg ^ 2. calculateMOIAboutCG on the other hand is delegated to the subclass Rectangle, Circle and Polygon. Although each shape's MOI could be calculated based off poly's method, it is far more efficient to have Rectangle and Circle separate where they can call an O(1) method for calculating versus poly's O(n) method where n is the number of vertices.

To Do List



This manager was created in Java with an Observer design patter using OOP practices.

The application allows the user to create, delete and edit categories. Categories are there to help organize the task list such that suers can easily deteminr the most important task to the least importance by assigning a priority if wanted. User can also have multiple task list so that they can seperate certain tasks such as a list for school activities and a list for errands. Each list contains individual task with the start/end date and details of the task.
The TaskListManager observers a TaskList which observes a Task. If a Task is changed(add, delete, edit) then only the list area gets updated. If a TaskList is (add, delete, edit) then the tabs at the top change appropriately. When the fields for a task are entered, they fields will only update if the fields are valid, otherwise the data remains static so the user is able to edit any mistakes.

Ecosystem Simulator



This simulator was created in Java with an MVC design patter using OOP practices.

Animals have ranks, based on their rank they act differently. An animal can be a Low, Middle, or High ranked animal. Each animal has a wait time for breeding, a time span which they must eat a meal otherwise they starve to death, an age limit, a color and a symbol. In this simulation animals of lower rank live longer, but are incapable of eating, therefore they will always die either from old age or being eaten. Low tier animals at the start of the round will always try to breed first if they can, otherwise they will move to a random location. Middle tier animals at the start of the round will try to breed, eat then move. High tier animals at the start of the round will try to eat, breed then move. Animals will only ever do one action per round and if one dies from starvation, then they will be left on the board until the round is fully complete to allow animals to eat the corpse.
